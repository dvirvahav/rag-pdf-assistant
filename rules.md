# Project Rules

## Commit Types
- feat: new feature
- fix: bug fix
- docs: docs update
- style: formatting only
- refactor: code change without new feature
- test: test changes
- chore: config or maintenance
- perf: performance change
- build: build tool change
- ci: CI pipeline change

## Branch Names
Use short names like: feat/login-page, fix/button-bug.

## Code Style
Follow project config files like .editorconfig.
